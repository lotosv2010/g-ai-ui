# 样式定制

`@g-ai-ui/ui` 采用 Tailwind CSS 构建。推荐将样式定制拆分为三层：

- 组件调用层：通过 `className` 做局部扩展
- 项目主题层：在 `tailwind.config` 中维护设计 token
- 全局策略层：通过 CSS 变量支持品牌主题与模式切换

## 1) 组件调用层（最低成本）

```tsx
import { ChatBubble } from '@g-ai-ui/ui'

export default function Example() {
  return (
    <ChatBubble
      message="已接收请求，正在生成处理方案。"
      isUser={false}
      className="mb-3"
    />
  )
}
```

适用场景：页面级微调、局部布局适配。

## 2) 项目主题层（推荐）

```javascript
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        brand: {
          50: '#f5f8ff',
          100: '#e6edff',
          500: '#3b82f6',
          600: '#2563eb',
          700: '#1d4ed8',
        },
      },
      borderRadius: {
        panel: '14px',
      },
    },
  },
}
```

适用场景：统一品牌色、圆角、间距体系。

## 3) 全局策略层（主题切换）

```css
:root {
  --chat-user-bg: #2563eb;
  --chat-user-text: #ffffff;
  --chat-ai-bg: #e5e7eb;
  --chat-ai-text: #111827;
}

.theme-dark {
  --chat-user-bg: #1d4ed8;
  --chat-user-text: #f8fafc;
  --chat-ai-bg: #374151;
  --chat-ai-text: #f3f4f6;
}
```

适用场景：多品牌、多主题、多租户。

## 实践建议

- 将颜色、圆角、阴影等关键变量沉淀为 token，避免散落在页面代码中。
- 避免直接改组件源码优先，先通过外层样式系统完成定制。
- 在设计评审中明确“组件默认样式”和“业务皮肤样式”的职责边界。
