# 样式定制

`@g-ai-ui/ui` 默认采用 Ant Design X 风格的浅色卡片体系。推荐将定制拆分为三层。

## 1. 组件调用层（低成本）

通过 `className` 做局部调整：

```tsx
import { Bubble } from '@g-ai-ui/ui'

export default function Example() {
  return (
    <Bubble
      role="assistant"
      content="已接收请求，正在生成处理方案。"
      className="mb-3"
    />
  )
}
```

## 2. 项目主题层（推荐）

在 `tailwind.config.js` 中维护 token：

```js
module.exports = {
  theme: {
    extend: {
      colors: {
        brand: {
          500: '#1677ff',
          600: '#0958d9',
          accent: '#36cfc9',
        },
      },
      borderRadius: {
        panel: '22px',
      },
      boxShadow: {
        panel: '0 18px 45px rgba(15,23,42,0.08)',
      },
    },
  },
}
```

## 3. 全局策略层（多主题）

通过 CSS 变量维护品牌策略：

```css
:root {
  --ga-primary: #1677ff;
  --ga-accent: #36cfc9;
  --ga-surface: #ffffff;
}

.theme-enterprise {
  --ga-primary: #0052cc;
  --ga-accent: #00b8d9;
}
```

## 建议

1. 优先调整 token，不直接修改组件源码。
2. 先在 Playground 验证对比效果，再回写业务项目。
3. 视觉变更与交互变更分批提交，降低回归风险。
